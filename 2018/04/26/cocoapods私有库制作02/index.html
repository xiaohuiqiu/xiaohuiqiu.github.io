cocoapods 私有库制作（第二篇）

发表于 2018-04-26 | 分类于 iOS | 阅读次数:

代码分层
代码库里面的结构也可以分层次通过文件夹来区分，文件夹可以多层嵌套。

如果podspec配置文件里面有依赖私有的库，那么本地验证的时候也要加上私用仓库地址

如果是验证远程版本库

其实实际使用时，我们可以不用执行验证的命令行。直接执行添加到远程版本库的命令就行，它会默认先执行验证的命令

--allow-warnings 表示忽略警告。 要加上这句，不然会添加不成功。
版本库更新
首先git提交，打tag再提交。

然后执行添加到版本库的命令。

Error

  • ERROR | [iOS] unknown: Encountered an unknown error (Unable to find a specification for BaseServer depended upon by HomeModule) during validation.
    如果有的用到私有库，验证的时候需要加上私有仓库地址。

  • ERROR | [iOS] unknown: Encountered an unknown error (There is a circular dependency between BaseServer/Manager and BaseServer/Common) during validation.
    依赖性不能是循环的 比如 a依赖b。b依赖c。c依赖a。这样不行。
